# IDEAS

Ideas to implement in folderjpg, organized by category.

--------------------------------------------------------------------------------
## A. New image formats

--------------------------------------------------------------------------------
### A1. Support for PNG, WEBP, BMP, TIFF

Extend the supported input formats beyond `.jpg`. Also search for `folder.png`, `cover.png`, `front.png`, `folder.webp`, etc. Magick.NET already supports them — just extend the search pattern.

--------------------------------------------------------------------------------
### A2. Extract cover art from MP3/FLAC files (ID3 embedded art)

For music folders, extract the cover art embedded in the ID3 tag of the first `.mp3` or `.flac` file when no `folder.jpg` is present. Requires an additional library (TagLib#).

--------------------------------------------------------------------------------
### A3. Extract cover art from PDF or EPUB files

For book libraries, extract the first page of a PDF or the cover image of an EPUB as the folder icon. Useful for Calibre collections.

--------------------------------------------------------------------------------
## B. New command-line options

--------------------------------------------------------------------------------
### B1. `--dry-run` — Preview without changes

Show what the program would do without writing any files. More accessible than the current internal DEBUG mode. Example: `folderjpg --dry-run "C:\Music"`.

--------------------------------------------------------------------------------
### B2. `--clean` / `--undo` — Remove generated icons

Traverse a directory and remove all `folderjpg-*.ico` files and the `desktop.ini` files generated by the application, restoring folders to their original state. Essential option for reverting changes.

--------------------------------------------------------------------------------
### B3. `--force` — Force regeneration

By default, if a `desktop.ini` already exists the folder is skipped. With `--force`, regenerate the icon even if it already exists. Useful when `folder.jpg` has been updated.

--------------------------------------------------------------------------------
### B4. `--depth N` — Limit recursion depth

Limit how many subdirectory levels the program descends. Example: `folderjpg --depth 2 "C:\Music"` only processes up to two levels.

--------------------------------------------------------------------------------
### B5. `--exclude pattern` — Exclude directories

Exclude folders whose name matches a pattern. Example: `folderjpg --exclude ".git" --exclude "node_modules" .`. Support multiple `--exclude` flags.

--------------------------------------------------------------------------------
### B6. `--stats` — Statistics at the end

Show a summary when finished: how many folders processed, how many icons created, how many skipped, how many errors. All the information is already available during processing — just accumulate it and display it at the end.

--------------------------------------------------------------------------------
### B7. `--log file.txt` — Save output to a file

Write the program's output to a log file in addition to the console. Useful for processing large libraries and reviewing the result afterward.

--------------------------------------------------------------------------------
### B8. Multiple paths in one command

Accept more than one directory as argument: `folderjpg "C:\Music" "D:\Photos" "E:\Books"`. Currently only `args[0]` is processed.

--------------------------------------------------------------------------------
### B9. `--json` — JSON output

Structured output mode for integration with scripts or automation. Each processed folder is emitted as a JSON object with fields `path`, `status`, `icon`, `skipped`.

--------------------------------------------------------------------------------
### B10. `--newer` — Only if the image is newer than the icon

Regenerate the icon only if `folder.jpg` is more recent than the existing `folderjpg-*.ico`. Smarter than a full `--force`.

--------------------------------------------------------------------------------
## C. Image improvements

--------------------------------------------------------------------------------
### C1. Icon with rounded corners

`--rounded` option to apply a rounded-corner mask to the generated icon, giving a more modern style similar to iOS/macOS apps.

--------------------------------------------------------------------------------
### C2. Control of generated icon sizes

`--sizes 16,32,256` option to choose which resolutions to include in the `.ico`. By default 6 sizes are generated; for very large collections, generating only 256px reduces disk space.

--------------------------------------------------------------------------------
### C3. Add border or shadow to the icon

`--border` option to add a subtle border or shadow to the generated icon, improving visibility on light or dark Windows backgrounds.

--------------------------------------------------------------------------------
## D. Windows features

--------------------------------------------------------------------------------
### D1. `--watch` — Watch mode

Monitor a directory in real time with `FileSystemWatcher` and automatically apply icons to newly created subfolders. Example: `folderjpg --watch "C:\Music"`.

--------------------------------------------------------------------------------
### D2. `--list` — List folders with applied icon

List all folders within a path that have a `desktop.ini` generated by folderjpg (identifiable by the `folderjpg-*.ico` pattern). Useful for auditing.

--------------------------------------------------------------------------------
### D3. Windows context menu integration

Register an entry in the Windows Explorer context menu ("Apply folder icon") via a registry key. Requires a `--install-context-menu` / `--uninstall-context-menu` subcommand with administrator permissions.

--------------------------------------------------------------------------------
### D4. `--restore-icon` — Restore icon for a specific folder

Remove the `desktop.ini` and the `.ico` from a specific folder (non-recursive). Complements the recursive `--clean`.

--------------------------------------------------------------------------------
## E. Cross-platform compatibility

--------------------------------------------------------------------------------
### E1. macOS support — `.DS_Store` / `SetFile`

On macOS, apply the icon using `SetFile -a C` or the NSWorkspace API so that Finder displays the folder icon. Currently the code is Windows-only.

--------------------------------------------------------------------------------
### E2. Linux support — `.directory` files (KDE/GNOME)

On Linux, create a `.directory` file with an `Icon=` entry so that file managers like Dolphin (KDE) or Nautilus (GNOME) display the custom icon.

--------------------------------------------------------------------------------
## F. Configuration

--------------------------------------------------------------------------------
### F1. Per-project configuration file `.folderjpg.json`

Allow placing a `.folderjpg.json` in the root of the directory to be processed with options like `excludeDirs`, `imagePriority`, `iconSizes`, etc. The file overrides the default parameters.

--------------------------------------------------------------------------------
### F2. Global configuration at `%APPDATA%\folderjpg\config.json`

Read a global user configuration from `%APPDATA%\folderjpg\config.json` with customizable default values (language, maximum depth, exclusions).

--------------------------------------------------------------------------------
## G. UX / Terminal

--------------------------------------------------------------------------------
### G1. Colored terminal output

Use ANSI colors: green for successfully created icons, yellow for skipped folders, red for errors. Improve output readability especially in large collections.

--------------------------------------------------------------------------------
### G2. Progress bar

Show a progress bar or counter `[42/318]` while processing directories. Requires counting eligible directories in a first pass or using an indeterminate progress bar.

--------------------------------------------------------------------------------
### G3. Quiet mode `--quiet`

Suppress all output except errors. Useful for use in automated scripts where only the return code matters.

--------------------------------------------------------------------------------
### G4. Meaningful return code

Return a non-zero return code if there were errors (`Environment.Exit(1)`). Currently always returns 0, which makes it difficult to use in scripts.

--------------------------------------------------------------------------------
## H. Code quality (technical debt)

--------------------------------------------------------------------------------
### H1. Remove unused SkiaSharp dependency

The project references SkiaSharp but doesn't use it anywhere. Remove it from `.csproj` to reduce executable size and eliminate an unnecessary dependency. (See TODO 007/1003)

--------------------------------------------------------------------------------
### H2. Constants for file names

Extract `"folder.jpg"`, `"cover.jpg"`, `"front.jpg"`, `"desktop.ini"` to static constants in the `Program` class. Makes maintenance and possible future changes easier. (See TODO 011/1007)

--------------------------------------------------------------------------------
### H3. Separate logic into classes

Split `Program.cs` (currently ~660 lines) into independent classes: `IconConverter`, `DirectoryProcessor`, `DesktopIniWriter`, `IconCacheRefresher`. Improves testability and readability.

--------------------------------------------------------------------------------
### H4. `--self-update` — Automatic update

Check if a newer version is available on GitHub Releases and download it automatically. Requires access to the GitHub API or a version endpoint.
